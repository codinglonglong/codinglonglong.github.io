    Could not open a connection to your authentication agent

    龙龙

    2017-01-09 22:11

    在执行ssh-add时，如果出现“Could not open a connection to your authentication agent”这一错误信息，可以通过如下方法解决：

    >> eval `ssh-agent`

    注意 是 ` 而不是 ' 。

    管理多个git账户

    龙龙

    2017-01-09 21:28

    因为博客用了两个github账户，一个放文章，一个放图片，所以出现了两个git账户同时存在的问题。这时可以通过如下的方法解决冲突：

    1. 生成两个ssh key.
    >> ssh-keygen -t rsa -C "email address 1"
>> ssh-keygen -t rsa -C "email address 2"
    1. 添加上一步生成的两个ssh key.
    >> ssh-add ~/.ssh/id_rsa1
>> ssh-add ~/.ssh/id_rsa2
    1. 创建配置文件.
    >> touch config

    # github1
Host github1
     HostName github.com
     PreferredAuthentications publickey
     IdentityFile ~/.ssh/id_rsa1
# github2
Host github2
     HostName github.com
     PreferredAuthentications publickey
     IdentityFile ~/.ssh/id_rsa2
    1. 测试.
    >> ssh -T git@github1
>> ssh -T git@github2

    注意

    1. 过程之中遇到yes或者no，都输入yes。
    2. 生成的ssh key别忘了放进github的“设置”里，并且注意结尾不能有回车。

    回到最初的起点

    龙龙

    2017-01-09 16:05

    时间才刚刚过去半年，一切感觉都不对了。没怎么碰代码的自己，在过去的半年里，除了一个字幕生成器以外，几乎没有什么完整的代码。虽然毕业以后，一直在忙碌工作的事情，适应新的环境，还出差学习了视频后期的基础知识，但不写代码的日子，总是在心里缺少些什么。少了刚上大学时“每天一个程序，每个程序都是传奇”的自信，少了争论哪种编程语言最好用时的面红耳赤，少了折腾代码解决bug时的那种驱动感。不过，单纯的我单纯地相信“只要努力，什么时候开始都不晚”，2017年，刚刚忙完教师证，买了新的笔记本，回到最初的起点，重做自己的技术梦。不管以后走不走开发这条路，有一颗热爱技术的心总是好的，因为热爱技术的人往往很可爱~~就像我们几个可爱的小伙伴…………
