    百度网盘大文件普通下载

    龙龙

    2017-06-15 02:02

    使用百度网盘下载大文件时，会提示安装客户端，关键是注册还需要手机号码，这对于个人隐私来说，总觉得存在隐患。因为下载文件本身和手机号码间没有必然的联系……

    也实在感觉到，随着互联网的发展，越来越多根本与手机号码无关的网络服务都在要求使用手机号码注册，最大的“理由”就是找回密码，这也难怪能接到各种陌生电话的骚扰……因为谁也不知道这个号码到底是谁泄露出去的……所以，到头来，一些受苦受难的普通人，只能选择不注册……因为没有你的服务，地球一样可以转……不排除，确实有人不在乎骚扰……

    对此，我专门进行了实验，两部手机，两个号码，一个专门用来注册日常需要使用的APP和网站，包括填写网购时的快递信息等等；另一个几乎什么都没有注册过……

    一年过去了，后者安静地，让我感觉到，我的手机主要功能是用来叫我起床，还有写备忘。

    值得一提的是，任何APP或者网站，在注册时都会告诉我们——此号码只用于找回密码……

    不过话说回来，当我们有着急的事情，需要临时下载一些文件时，该怎么办呢？

    可以使用Firefox浏览器 http://www.firefox.com.cn/ ，在文件下载页面右击，选择“查看元素”，选择“控制台”，在最底下“>>”后面输入：

    Object.defineProperty(Object.getPrototypeOf(navigator), 'platform', {get:function(){return '';}});

    再去点击“下载”按钮，安装客户端的提示就没有了。

    附加说明

    本文不针对任何索要手机号码的网络服务，只表示对互联网行业现状的担忧，因为地球还是在转的。

    import paho.mqtt.client as mqtt
import json

def on_connect(client, userdata, flags, rc):
    client.subscribe("test")

def on_message(client, userdata, msg):

>> cd mosquitto-1.4.11

>> make binary
>> make install
>> vi /etc/profile

    安装Deepin和RemixOS双系统

    龙龙

    2017-05-21 12:08

    RemixOS是PC版的Android，可以运行一部分Android应用。

    RemixOS下载地址： https://sourceforge.net/projects/remix-os/

    1、在Windows上，下载RemixOS安装包，解压缩，用Remix_OS_for_PC_Installation_Tool制作RemixOS安装U盘。

    2、将安装U盘插入Deepin所在的笔记本（基于Ubuntu的发行版同理）。

    3、启动Deepin，在根目录下创建文件夹Remix（这里需要管理员权限）。

    4、在Remix中创建目录DATA（用来存放数据）。

    5、把U盘上的内容全部拷贝到Remix目录中。

    6、修改引导文件。

    >> sudo gedit /etc/grub.d/40_custom
#!/bin/sh
exec tail -n +3 $0
# This file provides an easy way to add custom menu entries.  Simply type the
# menu entries you want to add after this comment.  Be careful not to change
# the 'exec tail' line above.
menuentry 'Remix OS For PC' --class android-x86{
    set root=(hd0,3)
    linux /Remix/kernel root=/dev/sda3/Remix androidboot.hardware=remix_cn_x86_64 androidboot.selinux=permissive quiet DATA=/Remix/DATA SRC=/Remix nomodeset
    initrd /Remix/initrd.img
}
>> sudo grub-install 40_custom
>> sudo update-grub
>> sudo reboot

    经过反复试验，RemixOS总是卡在开机的Logo画面上，两年前，我在同一个笔记本上安装RemixOS时是正常进入系统的。这次放了两个小时都没有进去，也没有任何提示信息……目前估计是系统镜像的问题，网上提到同样问题的人也很多……当前系统的版本是B2016112101。

    如果有哪位大神解决了这个问题，可以顺便告诉我一下，非常感谢。

    联系方式在“联系我” https://codinglonglong.github.io/posts/lian-xi-wo/ 页面中。

    注意

    1、(hd0,3)指的是第0号硬盘的第3号分区。

    2、引导文件不要轻易加空格。
